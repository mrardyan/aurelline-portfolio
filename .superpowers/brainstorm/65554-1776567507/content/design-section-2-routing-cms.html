Design · Section 2: Routing + CMS Abstraction

How pages connect, and how the CMS layer keeps Sanity swappable

🗺 Route Map

router.tsx
├── /                    HomePage ← all portfolio sections
├── /case-studies/:slug   CaseStudyPage ← detail view
├── /works/:slug         WorkPage ← portfolio item detail
├── /blog                BlogPage ← post listing
├── /blog/:slug          BlogPostPage ← single post
└── /contact             ContactPage ← inquiry form

React Router v7 is already installed. router.tsx exports a single createBrowserRouter — main.tsx renders it wrapped in providers.

🔌 CMS Abstraction Layer (lib/cms/)

One TypeScript interface, two adapters. Components never import Sanity directly.

index.ts — Contract
interface ContentRepository {
  getCaseStudies(): Promise<CaseStudy[]>
  getCaseStudy(slug): Promise<CaseStudy>
  getWorks(): Promise<Work[]>
  getWork(slug): Promise<Work>
  getPosts(): Promise<Post[]>
  getPost(slug): Promise<Post>
}

export const cms: ContentRepository
// picks adapter from env var
sanity.ts — Adapter
// Sanity client + GROQ queries
import { createClient } from
  '@sanity/client'

export const sanityRepo:
  ContentRepository = {
    getCaseStudies: () =>
      client.fetch(groq`...`),
    ...
  }
static.ts — Fallback
// imports from data/
import { caseStudies }
  from '@/data/caseStudies'

export const staticRepo:
  ContentRepository = {
    getCaseStudies: () =>
      Promise.resolve(caseStudies),
    ...
  }
Swapping CMS = change one env var. Set VITE_CMS_ADAPTER=sanity for production, static for local dev without Sanity credentials. Pages call cms.getCaseStudies() — they never know which adapter is running.

📐 Types (types/content.ts)

interface CaseStudy { slug, title, category, metrics, desc, body, images }
interface Work       { slug, title, category, thumbnail, body }
interface Post       { slug, title, publishedAt, excerpt, body }
interface Client     { name, logo }
interface Testimonial { quote, author, role, company }
interface Service    { title, items: string[] }

Both adapters and all page components import from here — types are the contract shared across everything.
