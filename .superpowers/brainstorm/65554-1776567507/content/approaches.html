3 Approaches to Structure & Architecture

Pick the one that fits your goals — I've marked my recommendation

A

Minimal Patch

Fix the obvious Figma-export mess, extract data files, move the hook. Keep the src/app/ nesting as-is.

src/
  app/
    App.tsx
    components/ (sections/, ui/)
    data/ ← extracted static data
    hooks/ ← use-once-in-view
  assets/
  styles/

Pros

  • Low risk, fast
  • Minimal churn

Cons

  • No pages/routing layer
  • No CMS abstraction
  • Still outgrows fast
B ⭐

Standard React Structure + Repository Pattern Recommended

Full restructure with pages, a CMS-agnostic content layer, and clear separation of concerns.

src/
  pages/ ← Home, CaseStudy, Work, Blog, Contact
  components/
    sections/ ← portfolio sections
    ui/ ← shadcn primitives
    layout/ ← Navbar, Footer
  lib/
    cms/
      index.ts ← ContentRepository interface
      sanity.ts ← Sanity adapter
      static.ts ← local fallback / dev mock
  hooks/ ← useOnceInView, useCMS…
  types/ ← CaseStudy, Work, Post, etc.
  data/ ← static content (fallback)
  assets/
  styles/
  router.tsx ← React Router routes
  main.tsx

Pros

  • Swap Sanity for any CMS later
  • Pages layer ready for routing
  • Clear boundaries — easy to navigate
  • Types make content self-documenting

Cons

  • More upfront work than A
  • More files to move
C

Feature-Based Modules

Group everything by feature: features/blog/, features/case-studies/, etc. Each feature owns its page, components, types, and data queries.

src/
  features/
    blog/ ← page + components + types
    case-studies/
    works/
    contact/
  shared/ ← ui, hooks, lib/cms
  pages/ ← thin route wrappers

Pros

  • Maximum isolation per feature
  • Scales to large teams

Cons

  • Overkill for a one-person portfolio
  • More indirection to navigate
  • Features share too much to truly isolate

My recommendation: B. It's the standard for React + CMS projects of this scale — you get clean boundaries and a swappable CMS layer without the overhead of full feature modules. Click to select, then tell me in the terminal.
