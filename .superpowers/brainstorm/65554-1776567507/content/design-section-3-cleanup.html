Design · Section 3: Code Cleanup

Targeted fixes — nothing speculative, only things that improve the existing code

1 — Fix scrollToSection prop drilling

Before — drilled through 3 layers
App.tsx
 ↓ scrollToSection
Hero.tsx → Navbar.tsx (uses it)
 ↓ scrollToSection
Footer.tsx (uses it)
After — shared utility
// lib/scrollTo.ts
export function scrollToSection(id: string) {
  const el = document.getElementById(id)
  if (el && window.lenis) {
    window.lenis.scrollTo(el, { duration: 1.5 })
  } else {
    el?.scrollIntoView({ behavior: 'smooth' })
  }
}

// Navbar, Footer import directly
import { scrollToSection } from '@/lib/scrollTo'

Also properly types window.lenis via a declaration in vite-env.d.ts — removes the (window as any) cast.

2 — Strip "use client" directives

Found in App.tsx and hero.tsx. This is a Next.js server-component directive — it has no effect in Vite and misleads anyone reading the code. Both get removed.

3 — Switch to @/ imports everywhere

Before
import img from '../../assets/personal-picture.png'
import { Hero } from '../components/sections/hero'
import { scrollTo } from '../../../lib/scroll'
After
import img from '@/assets/personal-picture.png'
import { Hero } from '@/components/sections/Hero'
import { scrollTo } from '@/lib/scrollTo'

4 — Clean up package.json name

"name": "@figma/my-make-file" → "name": "portfolio". Figma Make export artifact.

5 — Remove / relocate Figma artifacts

src/imports/MacBookAir4.tsx — check if used. If yes, rename and move to components/. If no, delete.
src/imports/svg-kpnmpaf0cj.ts — machine-generated name. Rename to something meaningful or delete if unused.
components/figma/ImageWithFallback.tsx — rename folder to nothing, move to components/ImageWithFallback.tsx.

6 — Rename files to PascalCase

React component files should be PascalCase. hero.tsx → Hero.tsx, navbar.tsx → Navbar.tsx, etc. Hooks stay camelCase: useOnceInView.ts.
What we're NOT changing
  • shadcn ui/ components — generated code, leave as-is
  • Tailwind / CSS setup — already correct
  • Vite config — already has @ alias and figma asset resolver
  • Unused dependencies — out of scope, risk of breaking things
